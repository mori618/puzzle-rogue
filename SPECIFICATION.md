# Puzzle Rogue 詳細仕様書

本書は、Puzzle Rogue プロジェクトにおける実装済みの仕様、各種システムロジック、およびパラメータを詳細にまとめたものです。
本仕様書の記述は、現在（2026年4月）のシステムファイル（主に `App.jsx` / `PuzzleEngine.js` / `tokens.js` など）のコード挙動に基づいています。

## 1. 基本システムとコアループ概要

### 1.1. ゲームの基本フロー
ゲームループは以下のフェーズを繰り返して進行します。
1. **タイトル〜セットアップ画面**：開始オプションの選択（例：コンボ加速等の初期設定など）。
2. **パズルフェーズ（Puzzle Screen）**：盤面内のドロップを操作し、ターゲットスコア（目標コンボ数）の達成を目指す。
3. **ショップフェーズ（Shop Screen）**：パズルフェーズで獲得した「スター」を消費し、新たなトークン（スキル／パッシブ）の購入、枠の拡張、パッシブの売却、覚醒による強化などを行う。

### 1.2. フェーズ別クリア・終了条件
* **ターゲット達成（クリア）**
  * 各ターンごとの獲得コンボ数の累計（Cycle Total Combo）が、目標値（Target）に到達すると「クリア」となります。
  * 目標値は一定の初期値からスタートし、サイクルごとや購入等によってスケールします。
* **ゲームオーバー**
  * 規定の「Max Turns」を消費し切った時点で、Targetに到達していない場合は GameOver となり、現行のセーブデータはリセットされます。

### 1.3. プレイモードの仕様
* **通常プレイ**：サイクルの進行に伴い難易度が上がり、クリアしていくベースモード。
* **Puzzle Practice Mode (練習モード)**：
  * メイン画面から専用メニューで開始。
  * 目標値（Target）がなく、無限に操作テストが可能。初期状態（ターン1、所持トークンゼロ）で始まり、「Pure Mode」（特殊消去のボーナス計算などを無効にする純粋コンボモード）や、操作時間（例：10秒）の調整が可能です。
* **彼岸（Beyond）モード**：
  * 通常サイクルを規定回数（サイクル25）クリアすると到達する無限モード。独自のBGMに変化し、システム上限に挑戦するエンドコンテンツとして機能します。

### 1.4. セーブデータと保存のタイミング
現行のシステムは `localStorage` を用いてステートを保存しています。
* **オートセーブ**：ターン終了ごと、ショップ利用時、設定変更時など、主要ステート（所有トークン、ターン数、スター、サイクル到達数など）の更新に合わせて自動保存されます（キー名: `puzzle_rogue_save` および `puzzle_rogue_stats`）。
* **永続要素**：ゲームオーバーになっても、各種「累計統計（lifetimeTotalCombo 等）」や「最高到達サイクル」などの実績データは `stats` として引き継がれます。ただし、ショップで購入したトークンや覚醒強化（枠拡張、時の砂など）は、新しいゲーム開始時にリセットされます。

## 2. パズルエンジンの詳細仕様

### 2.1. 盤面の基本構成
* **通常盤面**：5行 × 6列
* **拡張盤面**：パッシブトークン「巨人の領域（Giant Domain）」所持中、またはエンチャント効果によって 6行 × 7列 に拡張されます。

### 2.2. ドロップの種類と特殊な挙動
ドロップには基本となる5属性＋ハートの計6種類が存在し、特定のトークン効果で以下のような「特殊ドロップ」が生成されます。
* **基本ドロップ**：炎(Fire), 雨(Water), 風(Wood), 雷(Light), 月(Dark), ハート(Heart)
* **強化ドロップ**：消去時のコンボ計算でボーナスを加算。トークンの強化発生確率に基づき、生成時に「+」マークが付与されます。
* **ボムドロップ**：消去時に周囲のドロップを巻き込んで消し、コンボではなく「ボムによる消去数」として特殊な倍率計算に反映されます。
* **リピートドロップ**：消去判定時に1回で消えず、2回目の消去判定で消える耐久力を持ったドロップ。
* **スタードロップ**：消去時にショップで利用可能な「スター」を確定で獲得。
* **虹ドロップ**：全属性として代用できるワイルドカード（カウント制で複数回利用可能）。
* **ムーブドロップ**：操作して移動させた距離（マス数）に応じて内部カウントが蓄積し、ターン終了時にコンボ数に加算されます。

### 2.3. 操作時間（Time Limit）の算出ロジック
基本となる操作時間は以下の要素の合算・乗算で決定されます。
1. **ベース時間**：一律5,000ms（5秒） + 「時の砂」による強化分（1レベルあたり1,000ms）。
2. **パッシブ加減算**：各種トークンによる延長（+N秒）や「呪われた力」・狂戦士エンチャント等による短縮（-N秒）。星の数に比例する延長効果もあります。
3. **特殊固定上書き**：バフ「刹那の呪縛」や「背水の陣」など、特定の状態異常やトークンによる**固定時間への変更（例：4秒固定）**は最優先で適用されます。
4. **特殊消し乗算**：「五星の印（5個消し）」等で獲得した「次ターン時間係数（Multiplier）」を最後に乗算します。

### 2.4. コンボとスコア（倍率）計算式
コンボ数の計算は「パズル盤面で成立した純コンボ」をベースに、以下の多数のレイヤーで補正されます。
* **リアルタイムボーナス加算**：
  * 形消しボーナス：L字、十字、1列（Row）、4個ちょうど（Len4）など、指定された形で消した瞬間にコンボが直接加算・乗算されます。
  * 属性別加算ボーナス：炎や水など特定色が消えるごとに固定のコンボ数が加算されます。
* **最終コンボ倍率（Multiplier）補正**：
  * [条件合致ボーナス] 4色以上/5色同時消し、12個以上消去などの条件を満たした場合の倍率上昇。
  * 蓄積された各種統計（累計コンボ数、最大コンボ数、これまでのハート消去数 等）に関連する乗算パッシブ。
* **スキップボーナス**：ターンスキップを行った際、獲得量に対して「時短の心得」トークン等の乗算が適用されます。

### 2.5. スカイフォール（落下）の確率制御
* **基本ウェイト設定**：各属性は均等なウェイト（通常1）で設定。
* **スキル/パッシブ補正**：「〜の目覚め」で特定属性のウェイトを上げる、「〜の静寂」でウェイトを0にする、等の処理。
* **偏食家（Picky Eater）/ 禁忌（Forbidden）の例外**：
  * 「偏食家」による特定属性の出現ゼロ化。
  * 「禁忌」トークンや「虚無の契約」による、落ちコン自体の一切の停止（No Skyfallフラグ）。これらは最優先で処理されます。

## 3. UI/UXの挙動仕様

### 3.1. 画面外タップ（Backdrop）によるクローズ挙動
モーダルやオーバーレイ状態となる画面について、背景（Backdrop）タップ時の挙動が以下のように実装されています。
* **SettingsScreen / StatsScreen / CreditsScreen**など：画面外の半透明黒（バックドロップ）領域をタップ・クリックすると、当該モーダルが閉じる仕様。
* **HelpScreen / StartOptionScreen**などの情報系画面：同様に外部タップでのクローズに対応。

### 3.2. アニメーションと視覚的フィードバック
* **ドロップ操作**：ドラッグ＆ドロップ時は `requestAnimationFrame` を使用し、Reactの再レンダリングをバイパスして直接DOM（`transform: translate3d`）を更新することで、モバイル環境でも滑らかな追従を実現。
* **ポップアップ演出**：スター獲得時（`+N`）やコンボ加算時、購入時のパーティクルなど、動的生成された要素が一定FPS・一定秒数でフェードアウトする専用のCSS/JS連携システムでフィードバック。

### 3.3. サウンド・BGM管理
* `SoundManager` シングルトンにより、画面別にBGM（タイトル、パズルゲーム、ショップ、ゲームオーバー、彼岸モード等）が自動的にクロスフェード・切り替わります。
* 効果音（SE）も個別のIDで管理されており、コンボ時（ピッチが段階的に上がる）、ドラッグ開始/終了、残り時間のカウントダウンなどに連動して再生されます。

### 3.4. ドラッグ＆ドロップ（DnD）の管理
* トークンの整理や、アクティブトークンとパッシブトークンの表示エリア（Tabs）内での並べ替え・移動、また売却エリアへのドロップによる売却機能が実装されています。

## 4. キャラクター・プレイヤーステータス仕様

※システム上、固定の「キャラクター（固有スキル・固定パラメータを持つ主人公データ等）」は存在せず、**初期トークンセット**や**永続統計値**がプレイヤーの特性を決定します。

### 4.1. プレイヤーの管理パラメータ
ゲーム内で管理されるメタパラメータは以下の通り。
* `stars`：ショップの通貨。盤面消去やトークン売却、スキップボーナス等で獲得。
* `target`：サイクルごとに達成すべき目標コンボ数。
* `turn` `maxTurns`：現在の手数と、各種バフ/デバフによって変動する最大手数の制限。

### 4.2. 永続的な統計データ（Stats）
* `currentRunStats`（現在のゲームプレイでの統計）および `stats`（累計）として、以下のデータが記録されます。
  * `lifetimeTotalCombo`, `lifetimeClears`, `maxComboAllTime`, 各種形消しの実行回数、特定ドロップ（ハート等）の累計消去数。
  これらのデータは特定のパッシブスキルの効果（「〜の累計数に応じてコンボが上昇」等）の計算時に直接参照されます。

## 5. スキル・トークン・エンチャントの詳細仕様

スキルの実態である「トークン」は、主に `src/constants/tokens.js` で定義され、種別（アクティブ/パッシブ/呪い）、レアリティ（★1〜3）等を持ちます。

### 5.1. アクティブ（Skill）の仕様
パズル中にエネルギー（Energy）を消費して発動する効果です。
* **主要アクション**：
  * `convert` / `convert_multi`：盤面上の特定色ドロップを別の色に変換。
  * `board_change`：盤面を指定された数色のドロップのみに全面変換。
  * `row_fix` / `col_fix`：特定の行や列を指定色に固定変換。
  * `enhance_color` / `spawn_star` / `spawn_bomb_random` 等：ドロップに特殊効果やマーカーを付与。
  * `op_time_boost` / `temp_mult`：一時的な操作時間延長やコンボ倍率ブースト系のバフ。
* **レベル仕様**：トークンレベル（1〜3）によって、消費エネルギーが減少するものと、生成・変換数が向上するものの2通りが存在。

### 5.2. パッシブ（Passive）の仕様
常に効果を発揮、または特定の消し方を行った際に自動発動する効果です。
* **主要アクション**：
  * `color_multiplier`：特定属性を含めて消すと倍率上昇。
  * `shape_bonus`：十字、4個、L字などの形消し達成時にコンボ数加算や操作時間延長。
  * `skyfall_bonus`：落ちコン発生時の無条件コンボ加算。
  * 各種 `chance`：天空から特定のドロップ（強化、ボム、虹など）が降る確率の向上。
  * 統計依存系：これまでの最大コンボや消滅させた呪い数に応じて恒久的に計算を強化。

### 5.3. 呪い（Curse）トークンの仕様
* デメリット効果（操作時間短縮、特定のドロップが出なくなる等）を持つ負のトークン。
* 解除条件（Cond: `skill_uses` など）が設定されており、規定回数条件を満たすと解除され、多くの場合、強力な報酬アイテム（レアパッシブ等）に変化します。

### 5.4. エンチャントの仕様
* ショップで購入できる「トークン自体への付与効果」。
* 同じエンチャントを重ね掛けすると統合されてレベルアップし、効果値が上昇します（例: スカイフォール加算、コンボ加算値アップ）。
* *注意*：特定トークン（「力の封印」等）によってエンチャント効果全体が一時的に無効化されるギミックが存在します。

## 6. ショップシステムとリソース管理

### 6.1. ショップの陳列ロジック
* `generateShop()` 関数により、所持品の構成や拡張枠に応じて以下のように生成されます。
  * **ベース陳列枠数**：基本的に数枠から始まり、パッシブトークン「陳列の極意」による枠の拡張が適用。
  * **セール枠（Bargain）**：「商談の極意」による半額商品の追加。
  * **レアリティ抽選**：基本確率に基づき抽選されますが、「招福の鈴」所持によって高レア（★2以上の）選出ウェイトが加算されます。
  * **種類別選出**：一定確率でスキル、パッシブ、エンチャント、呪い解除の儀式用アイテム等が混在。

### 6.2. トークン価格の動的変動
* 同種のトークン（アクティブ/パッシブ/呪いのカテゴリ単位）を多く所持しているほど、ショップに並ぶ新規トークンの基本価格が指数関数的に上昇（Growth Factor が乗算）します。

### 6.3. リロール（刷新）機能
* スターを消費して品揃えを更新。
* 実行するたびに次回の更新費用が設定された定数比率で上昇します。

### 6.4. 覚醒ショップ（Awakening Shop）
* プレイ中に獲得した「スター」を用いてアンロックできる、現在のプレイを大幅に強化するためのショップ内タブ。
* 追加トークン枠のアンロックや、パズル中の全レベルアップイベント購入、さらに「時の砂」（操作時間+）といった資産の確保を行えます。これにより、そのラン（周回）におけるプレイヤーステータスの大幅な底上げが図られます。
