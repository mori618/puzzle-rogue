# Puzzle Rogue (パズル・ローグ)

このプロジェクトは、React + Vite で構築されたパズルRPGゲームです。
パズル＆ドラゴンのようなパズル操作感に、ローグライクなトークン・ビルド要素を組み合わせています。

## ✨ 特徴 (Features)

- **直感的なパズル操作**: ドロップを自由に動かしてコンボを狙う、馴染みやすいゲームプレイ。
- **無限のビルド戦略**: サイクルごとにショップで「トークン」を購入・強化し、自分だけの最強ビルドを構築できます。
- **奥深いコンボシステム**: 単純なコンボだけでなく、特定の形（十字、L字、正方形など）で消すことで発動する「形状ボーナス」や、スキル・パッシブ効果によるコンボ加算・倍率アップを駆使してハイスコアを目指します。
- **ハイリスク・ハイリターン**: 落ちコンを停止させてコンボ倍率を大幅に上げるパッシブや、呪いトークン選択でリスクを取って報酬を得る戦略など、プレイスタイルの幅が広い設計です。
- **限界突破システム**: コンボが上限値（約21億）に到達すると「限界突破」イベントが発生し、最終ステージへのワープが解放されます。
- **彼岸モード**: サイクル25をクリアすると「彼岸」へ至るルートが開かれ、コンボ上限・目標値上限が撤廃された無限の戦いへ挑めます。

## 🎮 ゲーム仕様 (Game Mechanics)

### 1. ゲームの目的と流れ

プレイヤーは各「サイクル」の開始時に提示される**目標コンボ数 (Target Combo)**を、規定ターン数内に達成することを目指します。
**最終目標はサイクル25のクリア**です（目標値: 約42億）。

1. **パズルフェーズ**: 盤面のドロップを操作してコンボを組み、合計コンボ数を稼ぎます。
2. **目標達成**: 1サイクル中の合計コンボ数が目標に達すると、そのサイクルはクリアとなります。
3. **ショップフェーズ**: サイクルクリア後、ショップで「★（スター）」を消費して新しいトークンを購入したり、既存のトークンを強化したりします。**トークンにはレアリティ（★1 コモン 〜 ★3 エピック）があり、ショップ内ではレア度に応じた色の枠で陳列されます。**
4. **次のサイクルへ**: ショップを抜けると、より高い目標コンボ数が設定された新しいサイクルが始まります。

### 2. 目標値の増加 (Target Scaling)

目標値はサイクルごとに増加し、節目のサイクルで大きくジャンプアップします。

| サイクル | 目標値（概算） |
| :---: | :--- |
| 1 | 8 |
| 5 | 約 56 |
| **6（ジャンプ）** | **約 368** |
| 10 | 約 2,400 |
| **11（大ジャンプ）** | **約 14,600** |
| 15 | 約 153,000 |
| **16（超ジャンプ）** | **約 967,000** |
| 20 | 約 1,540万 |
| **21（絶望ジャンプ）** | **約 1億** |
| 24 | 約 8億 |
| **25（最終ジャンプ）** | **4,294,967,294 (限界値)** |

### 3. 限界突破 (MAX COMBO)

- 1ターンの合計コンボがゲーム内の**コンボ上限（2,147,483,647）**に到達すると「限界突破」ダイアログが出現します。
- **はい**を選ぶと即座に**サイクル25（最終ステージ）へワープ**します。
- **いいえ**を選んでも、以降のサイクルクリア画面に「**ラストステージへ挑む**」ボタンが残り続けます。

### 4. パズルルール

- **盤面**: 基本 **5行 x 6列** (パッシブ「巨人の領域」装備時 **6行 x 7列**)。
- **ドロップ (Orbs)**: 炎 / 雨 / 風 / 雷 / 月 / ハート の6種類。
- **消去条件**: 同じ種類のドロップを縦または横に3つ以上揃えると消える（パッシブ「双連の極意」で2個から消去可能）。
- **操作時間**: 基本 **5秒** (ドロップを移動開始した瞬間から計測)。パッシブ・エンチャント効果等で増減する。
- **ターン**: 1回のドロップ操作 = 1ターン。
- **サイクル**: 基本 **3ターン** 構成 (エンチャント「時の刻印」や呪い「不運の枷」等で増減可能)。

### 5. 特殊ドロップ (Special Drops)

通常のドロップに加えて、スキルやパッシブ効果によって以下の特殊なドロップが出現することがあります。

| 種類 | 説明 |
| :--- | :--- |
| **強化ドロップ (+)** | 消去時のコンボ加算ボーナスを高める。パッシブ「強化増幅」で効果が上昇する。 |
| **ボムドロップ (Bomb)** | 消去時や落ちコン処理中に爆発し、周囲のドロップを巻き込んで消去する。「爆発の種」スキル等で生成。 |
| **リピートドロップ (Repeat)** | 消去時にグループ全体が復活し、リピート数だけ繰り返し消えてコンボを追加発動させる。「循環の理」スキル等で生成。 |
| **スタードロップ (Star)** | 消去すると通貨の「★スター」を追加獲得できる。「星の創造」スキル等で生成。 |
| **虹ドロップ (Rainbow)** | 全属性のワイルドカードとして機能するドロップ。一定回数消去すると消滅する。「虹の創造」スキル等で生成。 |

### 6. コンボ計算式 (Combo Logic)

各ターンの「最終コンボ数」は以下の計算式で算出されます。

```math
最終コンボ数 = floor( (基本コンボ + ボーナス加算) × コンボ倍率 )
```

- **基本コンボ (Base Combo)**: パズルで実際に揃えたコンボ数。
- **ボーナス加算 (Bonus)**:
  - 形状ボーナス（固定値加算）
  - スキル/パッシブによる加算（例:「力の鼓動」+2/3/5）
  - エンチャント効果（例:「連鎖の刻印」+3）
- **コンボ倍率 (Multiplier)**:
  - 形状ボーナス（正方形消し等による倍率）
  - パッシブ効果による倍率（例:「七連の闘気」7コンボ以上で2/3/5倍）
  - 全消しボーナス (x2)
  - 呪いの数に応じた倍率乗算（「厄災の祈り」装備時）
  - 一部パッシブには「動的コンボ倍率の最大上限」があり、「神ノ理」で上限引き上げ・撤廃が可能。

通常モードでは最終コンボは**上限 2,147,483,647** でクランプされます。彼岸モードではこの上限が撤廃されます。

### 7. 特殊ボーナス (Special Bonuses)

#### A. 形状ボーナス (Shape Bonus)
特定の形でドロップを消すとボーナスが発生します。対応のパッシブを所持していると追加効果が付きます。

| 形状 | 条件 | 基本効果 | パッシブ装備時の追加効果 |
| :--- | :--- | :--- | :--- |
| **5個以上連結** | 5個以上つながる | コンボ加算 **+1** | **次手**の操作時間に乗算ボーナス<br>(五星の印: x1.5/2/3) |
| **L字消し** | 5個でL字 | コンボ加算 **+1** | コンボ加算に追加<br>(鉤十字の型: +3/6/9) |
| **十字消し** | 5個で十字 | コンボ加算 **+2** | **次手**の操作時間に乗算ボーナス<br>(十字の祈り: x2/3/5) |
| **4個消し** | 4個ちょうど | なし | コンボ加算に追加<br>(四連の術: +2/4/6) |
| **横一列消し** | 横全列消去 | なし | コンボ加算に追加<br>(横一閃: +5/10/15) |
| **3x3正方形** | 3x3の9個 | なし | **コンボ倍率**に乗算ボーナス<br>(四方の型: x2/3/5) |

#### B. 全消しボーナス (All Clear)
- **初期盤面全消し**: 最初の操作と落ちコンで、ターン開始時に存在した全ドロップが消えた場合、**最終コンボ数 x2**。
- **完全盤面全消し (Perfect Clear)**: 落ちコン処理終了後、盤面が完全に空になった場合、**最終コンボ数 x2**。
  ※ 両方満たした場合は「初期盤面全消し」が優先されます（重複なし）。

#### C. スキップボーナス (Skip Bonus)
目標達成後、残りのターンをスキップするとボーナスを獲得。
パッシブ「時短の心得」による倍率が適用されます（Lv1/2/3: x6/8/10）。

### 6. ショップ仕様 (Shop System)

サイクルクリアごとに商品が陳列されます（デフォルト構成）。

1. **レベルアップ枠 (1枠)**: 所持トークン1つをランダムにLvアップする「ランダム強化」(基本5★)。
2. **パッシブ枠 (3枠)**: ランダムなパッシブスキル。
3. **エンチャント枠 (2枠)**: エンチャント付与アイテム（覚醒ショップ解放後）。
4. **アクティブ枠 (4枠)**: ランダムなアクティブスキル。
   - **セール品**: 「商談の極意」所持で、半額の商品が追加されます（2/4/6個）。
5. **特殊アイテム**: サイクル数・確率条件で「呪いの契約」「神秘の昇華」などが追加されることがあります。

**価格の動的変化**: トークンの価格は所持数に応じて指数関数的に上昇します（成長率x1.5）。同じトークンを大量に集めるほど高額になります。

**枠の拡張**: 「陳列の極意」「魔道の極意」で枠数を増やせます。

**リフレッシュ**: スターを消費して商品を入れ替え可能。使用するたびに費用が **1.5倍ずつ上昇** します。なお、サイクル開始時・目標達成時にリフレッシュ費用はデフォルト値にリセットされます。

### 7. 覚醒ショップ (Awakening Shop)

ショップの特別な拡張枠です。以下の機能を★を消費して解放できます。

| 機能 | 説明 | 価格 |
| :--- | :--- | :--- |
| **エンチャントショップ解放** | ショップにエンチャントアイテムが陳列されるようになる | 解放費用あり |
| **トークン枠の拡張** | アクティブ・パッシブ枠をそれぞれ1枠ずつ追加（通常最大10枠 / 彼岸モード中は最大15枠）| 100→500→2000→10000→50000★ |
| **ランダムレベルアップ** | 所持トークンをランダムにLvアップする常設機能 | 買い切り |

### 8. トークンスロット (Token Slots)

- **初期スロット数**: アクティブ・パッシブそれぞれ **5枠**
- **最大スロット数（通常）**: 覚醒ショップで拡張すると最大 **10枠** まで増やせる（5段階）
- **最大スロット数（彼岸モード）**: 彼岸モード中はさらに拡張でき、最大 **15枠** まで増やせる

## 🚀 開発とデプロイ (Development & Deployment)

### 1. 依存関係のインストール
```bash
npm install
```

### 2. 開発サーバーの起動
```bash
npm run dev
```
ブラウザで `http://localhost:5173` などを開いてください。

### 3. ビルドとデプロイ
このプロジェクトは `gh-pages` を使用して GitHub Pages に簡単にデプロイできます。

**A. `vite.config.js` の確認**
`base` の設定が `'./'` になっていることを確認してください。

**B. デプロイコマンドの実行**
以下のコマンドを実行するだけで、プロジェクトがビルドされ、自動的に `gh-pages` ブランチにプッシュされます。

```bash
npm run deploy
```

**C. GitHub Pages の設定**
1. GitHub リポジトリの **Settings > Pages** を開きます。
2. **Build and deployment > Branch** で `gh-pages` ブランチの `/(root)` を選択して保存します。
3. 公開されるまで数分待ちます。

### その他のコマンド

| コマンド | 説明 |
|:---|:---|
| `npm run build` | プロジェクトを本番用にビルドします。(`dist` フォルダに生成) |
| `npm run preview` | ビルドされた内容をローカルでプレビューします。 |
| `npm run lint` | ESLintを実行してコードの静的解析を行います。 |
